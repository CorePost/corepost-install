# Plan for corepost-install

## Роль репозитория

Установщик и провижининг-компонент: регистрирует устройство, генерирует локальные конфиги, раскладывает секреты и подключает `corepost-preboot` и `corepost-agent`.

## Текущее состояние

- Репозиторий создан заново.
- Функции install сейчас размазаны по старому `corepost-initramfs`, но полноценного установщика нет.

## Что сделать

- Выбрать форму поставки:
  - CLI на bash;
  - набор скриптов для Debian.
- Реализовать процесс установки:
  - регистрация устройства на сервере;
  - получение `deviceId`, секретов и политик;
  - генерация конфигов для preboot и agent;
  - установка systemd unit-ов и ранних boot hooks;
  - настройка TLS/CA;
  - опциональный экспорт параметров для mobile-клиентов.
- Подготовить безопасное хранение секретов на хосте и права доступа к ним.
- Добавить сценарии `install`, `reconfigure`, `uninstall`, `rotate-secrets`, `recover`.
- Описать, как installer взаимодействует с Debian VM в QEMU и как проверяется идемпотентность.
- Подготовить README и operator runbook.

## Критерий готовности

- Чистая Debian VM поднимается установщиком без ручного раскладывания конфигов.
- Один и тот же installer может подготовить устройство к preboot и post-boot агенту.
